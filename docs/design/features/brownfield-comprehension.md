# brownfield-comprehension — comprehension seeding of the System Map

**Status:** designed.

The shared dependency of every brownfield intake: pointing the loop at an existing
repo seeds a fingerprinted System Map, demand-driven — comprehension of the existing
system is paid **once**, at Design time, and encoded so downstream agents read a
cheap contract instead of re-reading the codebase (the founding intent's amortize-
comprehension principle).
