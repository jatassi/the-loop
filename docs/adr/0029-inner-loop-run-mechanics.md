# ADR-0029 · Inner-loop run mechanics: self-booking phase agents, per-task fold-in, typed halts, and the booking toolkit

**Context.** Building `inner-loop-workflow` forced the mechanics ADR-0001/0008 left open: who performs the bookkeeping ADR-0026 assigned to "one boundary step" (the script is a pure-orchestration sandbox per ADR-0012; ADR-0028 barred the validator from docs), how the blind deriver's injection payload crosses the no-filesystem workflow edge (ADR-0004), what a crashed or budget-killed run leaves behind, and what the harness realities are — agent death returns `null`, a budget ceiling throws at the next spawn, and plugin-agent `agentType` resolution from workflow scripts is undocumented. Resolved by grilling, 2026-07-02; sharpened by a fresh-context reader test the same day. Amends ADR-0026 (fold-in timing), ADR-0028 (validator capability envelope + verdict values), ADR-0009 (lifecycle step 2 — persisting results moves in-run; the session persists only human decisions at fold-back), and annotates ADR-0004 (args payload).

**Decision.**

- **Self-booking phase agents; no scribe.** The agent that ends a feature's run-participation **books** that ending — durably, on the [[integration target]], within its own spawn. The plan agent commits the plan artifact and flips `designed→planned`; on bounce it books the park. Each build agent books its own task (below); a *blocked* build agent books the park (still committing nothing on the branch — the booking rides the target). The validator books validate-or-park post-verdict: validations append, probe-pack pin, graph flip, Ledger re-render, escalation record on deviation — plus the perfect-verdict squash-merge it already owned. Every booking that flips graph status re-renders the Ledger in the same commit. *Amends ADR-0028's "writes no docs":* the ban now reads **no writes to the judged tree during validation** (the declared-mutation invariant, unchanged); post-verdict bookkeeping cannot influence the verdict it records. Conventions: bookings are **separate target commits** (on perfect, the squash-merge commit carries the code, then one booking commit carries the bookkeeping — ADR-0026's one-code-commit-per-feature holds); every agent **leaves HEAD on the integration target**; the parking agent **authors the recommendation menu** (menus are judgment — the agent with the failure context writes them into its return and the escalation record). Half-booked crash windows heal at re-entry, never re-execute: a build agent finding its task's commit already on the branch but the plan still `pending` re-folds from that commit (noting the reconstruction — deviation prose is lost, footprints are not); a validator finding the feature's squash commit landed but the graph unflipped completes the booking with a reconstruction-marked validations entry citing the squash commit.
- **Per-task fold-in** (*amends ADR-0026's fold-at-validate-or-park*). Each build agent, after its one branch commit, folds its own completion report on the target (`spine plan report`; task status `pending→built`; the first task also flips the feature `planned→building`, keeping ADR-0026's branch-inventory invariant readable mid-run). Crash cost stays exactly one task, re-entry state is readable from artifacts alone, and the validator reads builders' claims from the plan artifact as `agents/validate.md` already assumes — resolving that ordering inconsistency.
- **First block parks.** v1 runs tasks sequentially; the first blocked task ends the feature and the blocked agent books the park immediately. Within-feature park-and-drain is deferred to [[worktree-parallelism]], which builds the task-frontier machinery anyway.
- **Typed blocks; halts are run-level.** Every agent's blocked return carries a kind: **feature** (contract defect, semantic conflict → park, drain continues) or **environment** (dirty tree, test-harness or probe precondition down → the **run halts**, unbooked: the same condition would fail every subsequent feature identically). The kind decides booking inside the agent's own spawn: feature-kind books the park, environment-kind books nothing. The plan agent gains an environment-blocked return (it too can hit a dirty tree); a deriver `blocked` (missing prompt inputs) is environment-shaped — an args-construction defect. Budget exhaustion halts likewise — the harness throw fires at the next spawn, so in-flight agents complete and their bookings land; the session verifies a clean tree at the launch gate before the run ever starts. Exception policy for everything else: a non-budget exception or a schema-exhausted spawn is treated like a `null` return — the feature **stalls**, the run continues. **The `boundary-result` contract is amended (design_version 3→4):** `{ completed, parked (with menus), budget {spent, remaining}, halted?: {reason: budget-exhausted | environment-blocked, detail}, stalled: [{feature, phase, note}] }`. `stalled` records agent death (a `null` return: no retry, nothing booked — the graph never advanced, so the next stateless pass re-runs the phase) so the run never silently swallows a dead agent.
- **Remediation triggers only as sole blocker — and gets its own verdict value.** ADR-0027's round runs iff readiness is clean, forensics/spec/acceptance/runtime all PASS, the standards axis has findings (advisory-only still triggers — the round exists so craft findings get fixed), and the plan carries no remediation task. Because ADR-0028 makes `perfect` mandate the squash-merge while advisory findings must never fail a leg, the composition gains a third mechanical value (*amends the `validator-verdict` contract*): **`result: remediation-pending`** — all legs would-PASS ∧ standards findings nonempty ∧ round-marker absent; the merge is withheld and the pass-1 entry persists honestly at `docs/validations/`. **The remediation task is the durable round-marker**, appended by the validator's booking via a mechanical `spine plan remediate` (command-shaped, never freehand: footprint = flagged files; acceptance = each finding addressed or individually rebutted with evidence; **exempt from `plan check`'s criterion-coverage rules both ways** — it covers no feature criterion by design) and named in the validator's return so the script can spawn it. A fresh build agent executes it; re-validation **reuses the pass-1 expectation sheet** in-run (the contract didn't change — no second deriver spawn); across a crash the next pass simply re-derives — the deriver is cheap, blind, and stateless. The round-2 validator sees the marker and routes survivors by severity, composing plain `perfect|deviation`. Any other failure parks immediately with the round unburned — polishing craft on code the human may discard is waste.
- **`args` is the run's orientation snapshot** (*annotates ADR-0004*): `{ target, scope (a feature-id list, confirmed at the scope handshake), index (whole-graph statuses/edges/acceptance, no bodies), slices (per in-scope feature: the resolved node + contract bodies — the deriver's injection payload, keeping the blindfold prompt-fed and the deriver Read-only), plans (task summaries for re-entry: ids, status, depends_on, size), probe (the runtime-probe binding excerpt) }`, assembled by the session per `/the-loop`'s instructions. Run-scoped slices ride `args`; the whole design never does. State is in-memory thereafter; a crash re-seeds from artifacts — the stateless pass. **Frontier semantics:** runnable statuses are `designed` (enter at Plan), `planned` (enter at Build), `building` (resume at the first non-`built` task); a dependency is satisfied iff its feature is `validated` or `shipped`; dependent exclusion on park/stall/skip is transitive within scope; an in-scope feature that isn't runnable (parked, drifted, already validated) is skipped with a `log()` line, never an error. Because the script has no filesystem, **the plan agent's return is amended to carry task summaries** (ids, `depends_on`, size) — the script's only way to learn the task list of a plan written this run.
- **The booking toolkit.** Three new spine surfaces so no booking agent hand-edits graph YAML or Ledger prose: `spine set-status <feature-id> <status>` (parse→mutate→render, round-trip-safe), `spine ledger render` (deterministic projection: status counts, parked list from escalation records, frontier; preserves the hand-written "What this is" prose and the Run-history section — [[surfacing]] appends run entries; the first render **replaces** today's hand-written middle sections by design, retiring the CLAUDE.md hand-maintenance rule for loop-made commits), and `spine plan remediate` (above). To feed the render mechanically, the [[escalation record]] gains a structured block (new `escalation-record` contract in the design artifact: feature, phase, kind, deviation summary, recommendation menu, branch ref) atop its narrative. ADR-0006's never-disagrees guarantee becomes mechanical.
- **Spawning: `agentType` with plugin-local resolution; symlink fallback.** Every spawn passes a JSON Schema of the agent's documented return shape (validated structured output). The deriver spawns at `effort: low` (the one rigor dial built now); everything else inherits session model/effort. Features and tasks are sequential, so the v1 script is straight awaited `agent()` calls — `parallel()`/`pipeline()` arrive with worktree-parallelism. The build opens with two harness probes: whether plugin agents resolve as `agentType` from a workflow script (if not, the minimal configure leg of `/the-loop` **symlinks** the plugin's `agents/*.md` into the target repo's `.claude/agents/` — resolution guaranteed; the link tracks the plugin, so nothing goes stale), and the budget throw's error identity (so the script's catch distinguishes it from ordinary failures).
- **Script + test posture.** The script lives at `workflows/inner-loop.js`, launched by `/the-loop` via `scriptPath` + `args` — a sanctioned skill-instructed Workflow call. It is tested by a **shim harness** that executes the real script under `node:test`: the `export const meta` line neutralized, the body wrapped in an async function with stub `agent`/`parallel`/`pipeline`/`log`/`phase`/`args`/`budget`, scripted agent replies per path — asserting the `BoundaryResult` and the spawn sequence across happy, bounce-park, block-park, deviation-park + dependent exclusion, remediation, halt, and stall paths. No build step, no bundling: the shipped script is the authored script. Progress surfaces per feature via `opts.phase` groups, `label`s per spawn, and `log()` narration — the walk-away surface as composed (ADR-0019).
- **Amended 2026-07-04 (workflow-phase-grouping design).** Progress groups flip from per-feature to per-SDLC-phase: `opts.phase` carries `Plan`|`Build`|`Validate` (build and drive under Build; derive and validate under Validate), the feature id rides the label as the sole disambiguator, and `meta` declares the three phases as bare titles on its pinned single line. The `boundary-result` `stalled` entry renames `phase` → `agent` — same `agentType` value, finer-grained than the coarse phase opt and no longer a name collision with it (design_version 7→8).

**Why.** Self-booking keeps knowledge where it was produced — the agent that hit an outcome writes it, so durability is per-spawn and the crash quantum stays one agent — and it dissolves the scribe that ADR-0026's "boundary step" language implied without ever naming. Typed halts keep park-and-drain honest: feature-shaped failures drain the frontier, run-shaped ones stop the spend instead of manufacturing N identical parks. Sole-blocker remediation spends the one bounded round only where it can change the verdict. The args snapshot is the only way injection-enforced blindness (ADR-0028) crosses the workflow edge without handing the deriver a shell.

**Considered and rejected.** A dedicated boundary/scribe agent (a spawn per boundary whose only job is writing what the phase agent already knows; git authority is no less concentrated — every phase agent already commits); session-side bookkeeping at run end (parked state hides until return and a crash loses it all — violates ADR-0026's target-always-complete); fold-at-validate-or-park (reports ride script memory: a crash loses deviation prose, task statuses never durably flip, and the validator's prompt-fed claims contradict `validate.md` §0); within-feature drain on block (the blocked agent could no longer book its own park, and a re-plan likely discards the drained siblings); remediation whenever standards findings exist (a build+validate cycle on features that park anyway); remediation only on contract-force findings (re-opens the advisory-rot ADR-0027 rejected); index-only `args` (the Read-only deriver can't fetch, and a plan-agent relay fails on re-entry when Plan doesn't run); the full parsed design in `args` (the bloat ADR-0004 exists to prevent); copying agents into `.claude/agents/` (stale on plugin upgrade — the symlink keeps one source of truth); bundling `src/` modules into the script (a build step in a no-build repo; shipped ≠ authored); headless-only testing (slow, costly, nondeterministic — the probe binding already sheds those steps first).
