# ADR-0030 · Model selection: per-role bindings, decision-density build strata, and the visible session fallback

**Context.** The first self-hosted runs (2026-07-03) spawned every agent on the session model — Fable drove sonnet-grade build tasks at fable prices. ADR-0029's "everything else inherits session model/effort" was a placeholder posture, not a policy; the hand-run dogfood already picked tiers deliberately (build and validate on sonnet, derive on opus at low effort, plan on the session model). The spawn surfaces that need bindings: the four workflow roles (`plan`, `build`, `derive`, `validate` — derive's `effort: 'low'` hardcoded at the spawn), plus surface-declared session-side roles — plan's fresh-context audit (`agents/plan.md` §7), design's reader test and design-it-twice alternates (`skills/design/SKILL.md`), and repo-local agents like `landscape-survey`. Harness facts confirmed from the docs (2026-07-03): agent-definition frontmatter supports `model` (aliases, full ids, `inherit`) *and* `effort`; the per-invocation model parameter beats frontmatter; the session-side Agent tool has **no per-spawn effort**; `.claude/settings.json` tolerates unknown top-level keys (`additionalProperties: true` in the published schema); the sanctioned `userConfig`/`pluginConfigs` mechanism exists but requires marketplace installation this self-hosting repo doesn't use (agents are symlinked). One gap: the workflow `agent()` opts' precedence over frontmatter is undocumented. Resolved by grilling, 2026-07-03. Companion: ADR-0031 (the delegated executor the `via` field routes to).

**Decision.**

- **An open registry of dotted spawn roles.** Every spawn surface declares a role id — bare phase names for the workflow roles, dotted sub-roles elsewhere (`plan.audit`, `design.reader`, `design.alternative`, `build.rote|standard|complex`, `drive`). The registry is open: an unknown or unbound role is never a config error — it resolves to the **session-model fallback with visible provenance** (a `log()` line in the workflow, relayed at the run boundary; the acceptance criterion "visible, never silent" made concrete). Adding a surface never forces a config-schema change; the fallback line is the nudge to bind it.
- **Binding value `{ model, effort?, via? }`** (new `model-binding` contract). `model` is a Claude alias, a full model id, or the explicit value **`session`** — a deliberate inherit, distinct in provenance from an unbound role's fallback. `effort` absent inherits the session effort. `via` is the executor discriminator — `agent` (default, omitted in practice) or `grok-cli` (ADR-0031) — explicit so the workflow script routes agentType mechanically and delegation is visible in config, never inferred from magic model ids.
- **Two config layers, one resolver.** Defaults ship in the plugin at `config/model-bindings.json`; per-project overrides live under the namespaced `"the-loop".modelBindings` key in `.claude/settings.json`, personal ones in `.claude/settings.local.json` — the harness's own files, schema-tolerated, no parallel config system. A new **`spine models`** command merges defaults < project < local (whole-entry replacement per role) and prints the resolved table with per-role provenance (`default | project | local | fallback`). That resolver is the **single source for every surface**: the launch leg rides its output into `args.models`; Bash-holding agents (plan's audit) and session-side skills (design) invoke it themselves. Migration note: once real plugin installation exists (`configure-step-full` territory), the sanctioned `userConfig`/`pluginConfigs` path is the target.
- **Build stratifies by decision-density, not size.** Plan stamps **`tier: rote|standard|complex`** on every task contract — *how much the task leaves to decide*, the same rubric that picks subagent models in the AlphaMind dogfood — and the workflow routes `build.<tier>`. `rote` additionally requires correctness fully captured by the task's tests + lint: that clause is the delegation-eligibility rubric (ADR-0031), adopted as a **provisional starting point to be viewed with suspicion** — calibration-capture mines delegated-task outcomes later. `size` stays what it is (a sizing-gate footprint proxy); tier exists precisely because size mis-stratifies decision-dense-small and rote-large tasks. Task summaries and `args.plans` entries carry tier; a plan cut before this feature defaults to `standard` with `fallback` provenance; `spine plan check` validates the field.
- **Workflow plumbing.** Every spawn passes `model`/`effort` opts from `args.models[role]`; spawn labels carry the resolved model (`[sonnet] build:…` — the repo's `[Model] title` convention extended to the `/workflows` surface); derive's hardcoded `effort: 'low'` moves into the table. **Session-side limitation, recorded:** the Agent tool takes `model` only, so session-side roles honor `effort` only where an agent definition file carries it in frontmatter. **First build probe** (the ADR-0029 probe pattern): confirm empirically that workflow `agent()` opts beat agent-definition frontmatter — the one precedence the docs don't state.
- **The shipped default table** (observed rows from the 2026-07-03 hand-runs; unobserved rows are recommendations, flagged): `plan → session` (observed) · `plan.audit → opus` (fires exactly when a planning mistake is expensive) · `build.rote → sonnet` (haiku is the candidate once observed) · `build.standard → sonnet` (observed) · `build.complex → opus` (unobserved) · `drive → sonnet` (must tell truncation from defect) · `derive → opus @ low` (observed) · `validate → sonnet` (observed) · `design.reader → sonnet` (deliberate fidelity: the reader should misread the way a build agent would, and build is sonnet) · `design.alternative → opus`. Delegation ships **off**: `build.rote` binds to sonnet; a project opts in by rebinding it per ADR-0031.
- **`design_version` 4→5.** The `task-contract` contract gains `tier`; the `model-binding` contract is new. Both additive — no validated node drifts; plans already on disk default to `standard`.

**Why.** Model choice was the last silent policy in the engine — everything else that matters (verdicts, parks, fallbacks) is typed and surfaced, and per-spawn model spend is both the largest cost lever and a correctness lever (the AlphaMind eval shows tier mismatch corrupting outcomes in *both* directions). Per-role provenance and the logged fallback keep the perfection-bar ethos: the engine may inherit, but never invisibly. Decision-density beats the size proxy exactly on the corners that matter, and it folds delegation into ordinary binding — rebind a stratum, not flip a bespoke switch. One resolver for workflow and session surfaces prevents the two from drifting apart.

**Considered and rejected.** A closed role registry (every new surface forces a plugin-defaults edit; the visible-fallback machinery must exist regardless); executor inferred from recognizing grok model ids (delegation becomes an implicit property of a magic id list; a typo'd Claude id could silently classify wrong); flat string bindings (`'opus:low'`, `'grok:grok-build'` — a micro-syntax that doesn't extend to future per-role knobs); `userConfig`/`pluginConfigs` now (inoperable under the symlink install, enable-time prompting is wrong UX for a table, and the value would be a JSON-blob-in-a-string); bindings as a `docs/` artifact (machine config, not a design artifact — contradicts the harness-native posture, ADR-0016); size-keyed strata (`build.xs|s|m` — reuses a validated field but mis-stratifies on exactly the decision-dense-small / rote-large corners); a separate per-task `mechanical` boolean (subsumed by the `rote` tier — one axis, not two); agent frontmatter as the primary binding channel (invisible to config layering, unresolvable per project, and spawn opts override it anyway — it remains a carrier only where the Agent tool's missing effort parameter leaves no alternative).
