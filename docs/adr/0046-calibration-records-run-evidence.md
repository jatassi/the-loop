# ADR-0046 · Calibration records are run evidence — the automated capture commit is exempt from the no-bookkeeping rule

**Context.** ADR-0034 bans bookkeeping commits: everything in-flight is derived from git at run start, and committed status projections were v1's costliest ceremony. Calibration Memory (ADR-0007, designed as calibration-capture) needs per-run observations — agent counts, per-role token deltas, wall clock, re-slice events — that live only in ephemeral harness state and are unrecoverable once the session ends. It also introduces the loop's first fully automated commit to the target branch: the run's final `record` agent lands the calibration record with no human action.

**Decision.**
- **One capture commit per run** (`calibration: run <date>-<seq>`), landed by the run's final `record` agent. Records under `docs/calibration/runs/` are **permanent evidence** — the bug-corpus posture, never rewritten.
- **The exemption is scoped by the distinction that justifies it:** ADR-0034's ban targets *derivable* state (git already holds the truth; a committed projection can only go stale). Calibration observations are *underivable* — evidence, not bookkeeping. Anything derivable stays uncommittable: `docs/calibration/index.md` is wholly regenerated by `the-loop calibration-summarize` and carries no hand-maintained content.
- **The automation is bounded:** the record agent writes a script-computed payload verbatim plus git-derived enrichment, only under `docs/calibration/`, after all validators, and a capture failure can never alter the run summary.

**Why.** The no-bookkeeping rule's rationale (re-run derives fresh truth from git) simply doesn't apply to observations git never held; refusing the commit would discard the only evidence the founding "earns its context" thesis can be measured against. Scoping the exemption to underivable evidence keeps ADR-0034 intact — the derived index is exactly the kind of projection the ban exists for, so it stays regenerable, never source-of-truth.

**Considered and rejected.** Session-written capture at the run boundary (a manual step in disguise — depends on the session surviving and obeying); harness-hook harvesting (richer token data, but coupled to hook semantics around background completion and fires outside run boundaries); uncommitted machine-local storage (violates ADR-0007's in-target-repo posture and dies with the machine).
