# ADR-0032 · Adjust resolution mechanics: typed resolution kinds, the escalation-resolve command, and mechanical fold-back

**Context.** ADR-0009 fixed the surfacing lifecycle and ADR-0029 moved result-persisting in-run, leaving the middle un-built: nothing can *resolve* a park — `parked` is a one-way door. Designing `surfacing` (2026-07-03 grilling) forced the mechanics: what a human decision *is*, who folds it back, and how a retry survives ADR-0028's patch-id dedup. Amends ADR-0009 (lifecycle steps 3–5 mechanized; run history), ADR-0028 (dedup rule, waiver shape, a healing gap), and ADR-0029 (the booking toolkit grows).

**Decision.**

- **The adjust skill is the phase surface; two routes in.** `/the-loop`'s run-boundary relay hands off when `parked` is non-empty; the `resolve-parked` proposal routes here at re-entry. Docket first, then one escalation at a time in **graph order**, recommended-answer style; `stalled`/`halted` are relayed, never decisioned (nothing was booked for a stall; a halt's fix is outside the graph). Clean-tree gate at entry, mirroring the launch leg — tell, never reset.
- **Typed resolution kinds name re-entry points.** `retry` (same phase) | `fix-in-place` (Build) | `re-plan` (Plan) | `waive` (terminal; validate parks only) | `defer` (stays parked). **Pre-steps attach content without changing the kind**: research, a config rebind, a design amendment (the design skill, or `spine note` for a node-scoped directive), waiver recording. `amend-design` is therefore not a kind (it is re-plan with a pre-step), and neither is `drop` (a design amendment that removes the node). Menu options are kind-stamped at authoring — the `escalation-record` contract's menu becomes `[{resolution, option}]`, recommended first (`resolution`, because the record's `kind` already means feature|environment); parsers stay lenient to pre-amendment bare strings; the human may always go off-menu.
- **`spine escalation resolve <feature-id> <kind>` is the shared spine, and always the last act.** It validates the kind against the record's phase, flips status (retry: plan park → `designed`, build/validate park → `building`; re-plan → `designed`; fix-in-place → `building`, or `designed` on a plan park; waive → `validated`), performs kind-specific extras (re-plan deletes the plan artifact; retry-on-validate stamps the `retried` mark), deletes the record, and re-renders the Ledger. One booking commit per resolution — `<feature-id>: escalation resolved — <kind>` is the durable trace; resolutions get no Ledger line (git history is the archive, the record-deletion pattern). `--phase` is the damaged-park escape hatch (graph says parked, no record on disk — hand-edited graphs exist): validate the kind against the stated phase, skip the deletion.
- **`fix-in-place` is valid everywhere** — the human supplies the fix content; the loop resumes at the parked phase carrying it. Validate park: `spine plan fix` appends `fix-N` (`fix: true`, distinct from `remediation: true` so a fix never burns the feature's one remediation round; `covers: []`, exempt from plan check's coverage rules both ways; `depends_on` all prior tasks; defaults s/standard). Build park: the same, plus the blocked task resets to `pending`, chained behind the fix. Plan park: no plan exists to mutate — the directive lands as a feature-node note via the new `spine note <feature-id> <text>` (parse→mutate→render, `set-status` discipline), and the plan agent re-runs with it riding the injected slice.
- **Waivers are recordings; `waive` is the kind only when nothing is left to fix.** `spine validate waive` appends `{obligation, reason, approver}` into the feature's latest validations entry — **no expiry, ever** (the concept is retired; Ship's evidence package still lists live waivers on the frontier). A mixed decision — waive some findings, fix the rest — is `fix-in-place` with waivers recorded as a pre-step. The consuming half lands in `agents/validate.md`: a finding matching a recorded waiver (same feature, any prior entry) is recorded as waived, never counted toward the verdict — without it, fix-in-place-with-waivers re-parks forever. The `waive` kind squash-merges on human authority, mirroring the validator's perfect-path mechanics (message suffixed `— waived`); probe-pack pin iff the runtime leg PASSed; two commits (merge, then booking), the crash window between them healing at re-entry by the merge-message probe — never re-merging.
- **`retry` survives dedup by a durable mark.** Resolve's retry-on-validate recipe stamps the latest validations entry `retried: <date> — <reason>`. ADR-0028's dedup rule amends: judge the **latest** entry for the patch_id — unmarked → dedup-skip as today; marked → run all four legs fresh, the new entry consuming the mark by position (it is now latest and unmarked). A gap found en route closes with the same edit: dedup-skip on an unmarked **deviation** entry with the graph still short of `parked` completes the missing park booking reconstruction-style — the mirror of the perfect-path healing — instead of booking nothing, which left a permanently dead state (entry appended, park never booked, every future pass a no-op).
- **`re-plan` disposes; refs go last.** The resolve commit deletes the plan artifact (`designed` means "no plan yet" everywhere — args assembly, frontier semantics). The feature branch is **plain-deleted after the booking lands**: the human's explicit discard is the authority, validations entries hold the evidence excerpts, and the recoverability principle is hereby scoped to file deletions. Ref-deletion ordering is general — a crash can't give a deleted ref back, so refs follow the booking commit (waive's post-merge branch deletion likewise).
- **Run history is mechanized.** `spine ledger append-run` (run summary JSON on stdin → one deterministic newest-first bullet under `## Run history`), invoked by the session at **every** run boundary — troubled or not — in its own booking commit. Landing surfacing retires CLAUDE.md's hand-maintenance rule, per that rule's own sunset clause.
- **Notification stays one line.** If the notification-channel port is bound (harness-native default adapter), the boundary relay pushes a one-line summary when `parked`/`halted` is non-empty. Nothing more until ports-adapters-full.

**Why.** Kinds-name-re-entry keeps fold-back a table lookup while judgment stays prose — the parking agent authors options where the failure context lives (ADR-0029), the human decides, and nobody interprets free text into git operations. Resolve-as-last-act makes the record's deletion the closing event, so every crash window leaves the escalation honestly open. The retry mark exists because dedup and retry are the same event with opposite intent — only a durable, forward-readable human order distinguishes them.

**Considered and rejected.** Free-form resolutions interpreted by the session (fold-back becomes judgment; the booking discipline exists to prevent exactly this); a six-kind set with `amend-design` separate (mechanically identical to re-plan — the difference is a pre-step); `fix-in-place` restricted to validate parks (build parks get reset-and-chain, plan parks the node-note channel — the human's one-line answer shouldn't cost a full re-plan); waiver expiry (dead concept — permanent until a human deletes it, git archives); tag-then-delete branch archiving on re-plan (plain delete; the human's discard is the authority); blast-radius docket ordering (graph order is simpler and predictable); run-history lines for resolutions (the resolve commit message is the archive).
