# ADR-0028 · Validate: the validator protocol — four legs, mechanical verdicts, two-severity findings, and the probe pack

**Context.** Validate is the linchpin of the [[perfection bar]] and the hard gate the field has not armed: a 2026-07-02 landscape survey ([research findings](../research/2026-07-02-validate-landscape-survey.md)) found every comparable system keeps machine verdicts advisory, documents builder-gaming at 30%+ rates on susceptible tasks (F1), names "tests pass, feature doesn't work" as the central failure (F2), and ships **no** mechanical anti-test-tampering (O2) and **no** non-web runtime verification (O1). ADR-0013 fixed the posture (independent validator, three legs, probe as port); the mechanics, the verdict model, and four adopted brainstorm mechanisms (design.md validate-node notes, 2026-07-02) remained open. Resolved by grilling, 2026-07-02. This ADR amends ADR-0013 (protocol mechanics; the independence posture made concrete), ADR-0026 (conflict handling), and completes ADR-0027 (severity routing for standards survivors).

**Decision.**

- **Pipeline: a readiness stage + four legs, deterministic-first.** Stage 0 **readiness** (setup, not a leg — emits BLOCKED, never FAIL): clean tree, rebase onto the target tip, task-branch fold-in, preconditions (test harness runnable, probe runnable). Leg 1 **integrity forensics** (mechanical scan + triage). Leg 2 **conformance** (spec axis + standards axis, never merged, per ADR-0027). Leg 3 **acceptance** (the project's test harness). Leg 4 **runtime** (pack replay + new-feature exercise + delta proof, sharing one bring-up). A confirmed forensics hit **short-circuits** the run — a gamed diff poisons every downstream observation; any other failure still runs the remaining legs, because park-and-drain surfaces batched escalations and half an evidence package forces a second round-trip.
- **Conflicts: the union rule** (amends ADR-0026's "a rebase conflict re-parks"). A **trivial conflict** — every conflicted hunk resolves as the pure union of both sides, no tokens authored or discarded (the barrel/route-table case) — is resolved by the validator, recorded in readiness evidence, and validated by the downstream legs; a wrong union surfaces as a leg failure, never lands silently. Anything past the union line is a **semantic conflict**: resolution would be authoring → BLOCKED, parked as ADR-0026's conflict-shaped deviation. This is also [[worktree-parallelism]]'s trivial-merge relaxation, designed once.
- **Per-leg verdicts, mechanical composition.** Each leg returns **PASS | FAIL | BLOCKED | SKIP** — fail-closed (undecidable → FAIL: a false PASS ships broken code, a false FAIL costs one human look); BLOCKED means *could not observe* and routes an environment-shaped menu, never conflated with FAIL; SKIP always carries a reason and is **sanctioned** only when it traces to a recorded Design-time opt-out (ADR-0013's probe opt-out), else it is a cascade skip. **`result` computes: perfect iff readiness was clean (trivial resolutions allowed, recorded) and every leg is PASS or sanctioned-SKIP.** No judgment sits at the composition step; judgment lives inside legs 2 and 4 where it is unavoidable.
- **Findings carry exactly two severities, assigned by the citation test.** A [[finding]] is **contract-breaking** iff it cites the specific obligation it falsifies — an acceptance criterion, an interface-contract clause, a task-selected [[project standards]] rule, or an integrity rule — plus the falsifying observation; a finding that cannot cite an obligation is **advisory** (recorded, never parks). No middle tier: every tier boundary is a judgment call made under adversarial prompting (survey F3), and one mechanically-testable boundary kills the "validation flagged something" escalation fatigue. Routing: forensics → always contract-breaking; spec axis → severity routes immediately (no remediation — fixing spec is rebuilding, a human call); standards axis → all findings batch into ADR-0027's one remediation round first, then survivors route by severity — **a task-selected project standard has contract force; the shipped baseline catalog is advisory forever**; acceptance red → contract-breaking; runtime → criterion unmet or unexplained delta contract-breaking, incidental observations advisory.
- **A waiver is a resolution, never a verdict value.** The contract-breaking escalation menu gains a fifth option: **waive** — `{ obligation cited (never file:line), reason, approver, expiry?: date | feature-id condition }`. The verdict stays `deviation` permanently; the fold-back squash-merges on human authority with the waiver recorded. The target invariant becomes "only validated **or human-waived** work." A live waiver suppresses re-flagging of that obligation-violation on this feature only; expired-unresolved waivers surface as Ledger attention items; Ship's evidence package lists all waivers on the frontier. Machine verdict and human authority stay in separate columns so calibration reads clean signal.
- **Evidence contract.** Every leg carries **`evidence`** (captured output — commands, exit codes, bounded excerpts, probe transcript excerpts; **anti-fabrication:** never paraphrased from expectation), **`unobserved`** (the negative space, on PASSing legs too — what "perfect" did not cover), and per finding **`reobserve`** (the command/probe step that re-produces the observation, where one exists). Persistence keeps excerpts + reobserve commands, not raw logs — the durable path back to an observation is re-running it.
- **Blind derivation: two agents; the inputs are the blindfold.** A cheap **blind deriver** receives only the feature-level contract slice (feature node, criteria, interface contracts) + the probe binding — no diff, no builder tests, no completion reports, and **no plan artifact**: task contracts carry footprints (the implementation's file layout) and decomposition-shaped criteria, so handing them over would leak implementation structure through the blindfold, and task-level checking already lives in legs 2 and 3 (2026-07-02 refinement). It emits the [[expectation sheet]]: per feature criterion, the expected observable behavior and probe steps to elicit it, each falsifiable. The validator then receives the sheet plus everything else; completion-report deviations arrive strictly after derivation, as claims to verify. Blindness is enforced by injection, not instruction (survey F7/D10) — and by **control-group posture**: the deriver's surface is procedural only, never telling it that it is blind, what it is blind to, or that a validator judges against its sheet (a control participant is never told they're the control; disclosure hands it a role to perform instead of a procedure to follow). Divergence routes two ways: code fails the sheet → ordinary finding by the citation test; the *sheet* proves ambiguous/unfulfillable → a typed **spec-ambiguity** advisory folded back to the Design artifact, not just the verdict record.
- **Forensics: a deterministic scanner, then justified triage.** `spine validate scan` diffs the feature branch against its merge-base and cross-references the plan artifact + completion reports. v1 tripwires: existing-test mutation (edits/deletions inside pre-existing test code; additions fine); disabling directives added to existing tests; suppression directives in any dialect; harness/config tampering outside the declared footprint; test-environment sniffing in non-test source (test-detection patterns only); exit-code/evaluator manipulation; undeclared footprint excursion (`footprint_actual ⊄ expected ∪ declared deviations`). *Fixture-mirroring constants deferred to a scanner v2* — the noisiest tripwire, and the delta proof already catches its consequence. Hits are **presumed findings**: the validator triages each — **confirm** (contract-breaking, short-circuit) or **dismiss** with `{ tripwire, location, why this is not the banned move }`, recorded in leg evidence even on PASS, never silent, never a findings-channel entry (noise control). This is the one sanctioned downgrade of a mechanical signal — bounded by per-hit structure, mandatory justification, and audit visibility.
- **The behavioral diff decomposes; no full double-run.** **Pack replay** (the "nothing else changed" half): every pinned exercise from prior features must reproduce its pinned observations on the merged tree — a miss is contract-breaking *with a built-in citation* (the pinned expectation, carrying its feature-id). **Delta proof** (the "delta is real" half): the new feature's blind-derived exercise runs on both trees and must be **red on the merge-base, green on the merged tree** — watch-it-fail lifted to the probe level. Green-on-both is a **vacuous exercise**: contract-breaking, citing the criterion whose runtime-observability failed to discriminate. Two bring-ups total.
- **The probe pack.** `docs/probes/<feature-id>.md` (ADR-0021 named dir): narrative + a structured block of `{ action, expected observation }` steps. The validator *emits* the executed exercise (steps + captured observations); the **boundary fold-in writes the file** — bookkeeping stays with the boundary (ADR-0026), flags-not-fixes holds. Volatile fields masked at pin time so replays are judgment-free; AI re-enters only on mismatch. **Full-pack replay every validation** in v1 (packs are small pre-self-hosting; impact-scoping is speculative until calibration demands it) — and Ship's full-system check *is* full-pack replay, so Ship inherits content for free. Flake protocol: a failed step retries twice; consistent red → regression finding; intermittent → pass + one advisory + the entry marked `flaky` at fold-in (mark suppresses repeat advisories and stands as a Ledger attention item). **Supersession rule:** a replay failure is legitimate only when the new feature's contract citably names the changed behavior — then the fold-in re-pins; no citation → regression.
- **Verdicts persist append-only at `docs/validations/<feature-id>.md`** — one file per feature, every validation appended (initial, remediation, re-validate), never deleted: the calibration substrate (survey D7) that makes "which leg should have caught this escaped bug" answerable later. Entries carry `{ design_version, patch_id, readiness, legs (verdict, findings, evidence, unobserved), result, spec_ambiguities, waivers, exercise }`. **Patch-id dedup:** a request whose `git patch-id` matches the feature's latest entry returns that verdict instead of re-running. The boundary fold-in writes the file in the same bookkeeping commit as the graph flip, pack pin, and Ledger re-render; the Ledger stays a derived surface (latest result + open waivers + flaky marks).
- **Capability envelope.** Validator tools: `Read, Grep, Glob, Bash, Write, Edit` — **no Agent** (the deriver is orchestrator-spawned; judgment can't be delegated). Write/Edit's sanctioned use is readiness-stage conflict resolution under the union rule, governed by the **declared-mutation invariant**: every tree divergence beyond mechanical rebase replay must appear in readiness evidence as a declared trivial-conflict resolution — undeclared divergence is git-visible and auditable. Stated honestly: this is instruction + audit, not capability removal; the validator, unlike the builder, has no incentive gradient toward mutation. The validator performs the perfect-verdict squash-merge and branch delete (ADR-0026); it writes no docs.
- **Rigor dials, named for effort-level scaling** (actions.md), not built: deriver split (separate agent ↔ ordered reads ↔ none), triage depth (per-hit judgment ↔ auto-confirm), delta proof (on ↔ off, keeping pack replay), replay scope (full ↔ impact-scoped), headless-agent exercise steps (sparse ↔ none), deriver/validator model+effort tiers.
- **Self-hosting probe binding — the fixture-repo probe** (closes ports.md's TBD): bringUp = `bin/probe-fixture.js` (bin/, not test/ — Node's test runner executes every `.js` under `test/**`) creates a temp git repo seeded as a plausible target repo (variants: empty for cold-start, populated for spine exercises); exercise = shell steps driving `node bin/spine.js` as a user would (never in-process imports — that is leg 3's territory), with **headless harness invocations (`claude -p`) as ordinary shell steps** for agent-pack surfaces — sparse, aggressively normalized, first thing the effort dial sheds, recorded as the binding's known soft spot; teardown = remove the temp dir. The shape ("fixture + CLI-as-a-user + sparse headless-agent steps") is the general answer for CLI/tool projects — survey O1's unoccupied territory.

**Why.** The survey's evidence orders the priorities: F2 (agent-written tests pass while features fail — SWE-bench+ measured 31% weak-test passes) justifies runtime observation held away from the builder; F1/O2 (documented gaming, zero shipped defenses) justify the forensics leg; F3 (judge over-reporting) dictates the two-severity citation test and mechanical composition; F4 (noise kills authority) dictates triage and the advisory channel; F5 dictates pin-then-replay determinism; O5 (nobody arms a hard gate) is exactly why every mitigation here — BLOCKED≠FAIL, deterministic mapping, typed waivers, evidence with negative space, append-only calibration substrate — exists before the gate arms.

**Considered and rejected.** Three severity tiers (BMAD-style medium feeds a CONCERNS state we don't have; two judgment boundaries where one mechanical test suffices); WAIVED as a verdict value (poisons calibration; verdicts must stay machine-honest); hits-are-findings with no triage (this very repo's `agents/build.md` names banned directives in prose — doc edits would park features); agent-judged forensics without a scanner (the builder can't argue with grep; an agent it can); one validator with ordered reads instead of a deriver (instruction-enforced blindness degrades, F7); a live before-tree probe run of the full pack (the pinned expectations already encode the before-state; only the delta proof needs the second tree); impact-scoped replay in v1 (blast-radius inference before the loop has ever run is the speculation the constitution bans); a cross-feature validations index (greppable per-feature files + the derived Ledger cover it); waivers that generalize across features (a waived smell on A must never quiet B); denying the validator Write/Edit (union resolution via `merge-file --union` plumbing is fiddlier than the rule it implements; replaced by the declared-mutation invariant); the criterion-guided mutation audit and the validator-regret loop (deferred: the first needs cost data, the second needs Evolve — the verdict record is its day-one seam); fixture-mirroring tripwire in v1 (noise; the delta proof catches the consequence).
