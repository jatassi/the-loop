# ADR-0025 · Plan mechanics: gateless with compensating machinery, persisted task contracts, overlap-serialized footprints

**Context.** Building the `plan` feature forced the mechanics ADR-0011 left open: where the decomposition lives, what shape Build consumes, and whether a human approves it. A 13-system landscape survey (2026-07-01, session record) sharpened the stakes: a human plan-approval gate is the field's single most universal pattern (~11 of 13 systems), while our two core bets — proxy-based over-decomposition and bounce-to-re-slice — exist nowhere else. The survey also showed the field's direction of travel: mature systems are automating their plan gates away and compensating with mechanical checks and fresh-context validation (BMAD v6 deleted its Draft→Approved gate; Kiro shipped run-all only behind a validation gauntlet).

**Decision.**
- **No plan-approval gate.** The inner loop stays autonomous (ADR-0008); the human gates remain Design and Ship. In its place, the compensating machinery the gate-removers converged on:
  - **Mechanical checks** in `spine plan check`: criterion coverage both ways (every feature acceptance criterion claimed by ≥1 task, every task claims ≥1 criterion), overlap ordering, sizing classes, edge integrity, round-trip.
  - **Conditional fresh-context audit**: a separate agent, handed only the injected slice and the plan, adversarially re-derives and diffs — triggered by contract surface (multiple contracts touched), blast radius (multiple dependent features), or judgment that a planning mistake is expensive; not run unconditionally.
- **Plans persist** as per-feature artifacts at `docs/plans/<feature-id>.md` (named directory per ADR-0021): hybrid narrative + one machine-parsed `## Tasks` block, loop-written. Validate consumes the task contracts, calibration mines estimate-vs-actual, and the run boundary has a resting record to surface.
- **The task contract** (contract: `task-contract`) is the Plan → Build handoff: `id, title, status, covers, acceptance, injects, footprint, size, depends_on`. Its **return type is schema'd now** (contract: `completion-report`): Build folds `{result, footprint_actual, diff_actual, deviations, summary}` back into the task node — simultaneously the Calibration Memory feed and the mechanical trigger the bounce rule lacked (actual ≫ expected footprint = a re-plan/re-slice signal).
- **Overlap serialization** makes ADR-0012's file-disjointness constructive rather than absolute: tasks whose expected footprints intersect must be connected by a `depends_on` path (a hub file gets one owning task or a chain). Only *unordered* overlap is an error; parallelized tasks remain disjoint by construction.
- **Sizes are `xs | s | m` and nothing larger.** `m` is the comfort ceiling — allowed only with narrative justification (warned by the check); a task that can't get under it splits, and a feature that can't split **bounces with a reslice brief** (why irreducible + suggested re-slices), persisting the analysis the way Task Master persists its expansion prompts.

**Why.** The gate question is a posture question: a mid-run approval would make every feature's plan a human round-trip and hollow out the walk-away surface — while the survey shows even gate-first systems migrating to exactly the machinery we adopt instead. Persistence follows the artifact spine's own rule (durable state lives git-versioned in the target repo, ADR-0002/0021) and gives Validate, calibration, and crash recovery one addressable record. Schema-ing the completion report before Build exists is deliberate: the contract's consumer side (Validate, calibration) was designed first so Build has a fixed target.

**Considered and rejected.** A full plan-approval gate (breaks ADR-0008's autonomy for the field's most-automated-away pattern); ephemeral workflow-internal plans (the feature is the resumable unit per ADR-0009, but an unpersisted plan leaves Validate and calibration nothing to consume and the run boundary nothing to surface); strict file-disjointness with no relaxation (fails routinely on hub files — barrel exports, route tables, shared types — turning ordinary structure into escalations); unconditional fresh-context validation (a full agent per plan for mostly-small plans; the triggers scale the spend to the stakes).
