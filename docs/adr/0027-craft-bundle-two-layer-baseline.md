# ADR-0027 · The craft bundle: a two-layer craft baseline, per-task standards injection, two-axis validation with one bounded remediation round

**Context.** Nothing shapes *how* build agents design and write code beyond the lint gate (mechanical) and the build agent's process rules. The user wants architecture patterns, code standards, smells, and dev-approach guidance bundled. A 26-source landscape survey (2026-07-02, session record) established the field's consensus: linters own everything mechanically checkable; build-time context must stay lean (measured: indiscriminate context files cost ~+20% inference for ≈0 gain, and instruction compliance decays over a session); smell catalogs belong at review, not in the builder's window; the strongest delivery mechanisms are per-task standards selection persisted at plan time (Agent-OS), deterministic injection into subagents (no model-triggering gamble), and two-axis review that never merges spec fidelity with craft judgment (mattpocock PR #394). Resolved by grilling, 2026-07-02.

**Decision.**
- **Two layers, repo wins.** A universal **[[craft baseline]]** ships in the plugin (versioned with the engine); **[[project standards]]** live in the target repo at `docs/standards/` (durable, git-versioned, per ADR-0002). Precedence is one-directional and explicit: a documented repo standard always overrides the baseline — where the repo endorses a pattern the baseline would flag, the finding is suppressed.
- **The baseline is a port** (`craft-baseline`, optional tier — inventory row in ports.md). Default adapter: the plugin's bundled pack. Any pack honoring the capability contract (the three consumption points below) swaps in with one config line; unbound routes around — build agents then carry only project standards.
- **The bundled pack is three pieces, matched to insertion points** (never one mega-file — build context stays lean while review gets the full catalog):
  1. the **[[build constitution]]** — one page: the reuse-before-build ladder, completeness rules, banned reasoning moves, anti-speculation;
  2. **design vocabulary & principles** — deep modules, seams, leverage/locality, the deletion test — consumed at Design (shaping contracts) and Plan (interface thinking);
  3. the **review catalog** — smells as one-line leading words with observable tells — consumed by the validator's standards axis.
  Admission test for every rule in any piece: would the model or the linter already know this? Then it doesn't land.
- **Injection is deterministic, selection is planned.** The constitution is a mandatory read step in the build agent — every task, no triggering gamble. Everything else is selected per task **by the Plan agent** and persisted as a `standards: [path]` field on the [[task contract]] (amends the `task-contract` contract; design_version bumped 1 → 2). `spine plan check` validates the paths; `spine plan task` returns them in the builder's slice. Selection is visible in the artifact, auditable, and the validator knows exactly which standards a task was built under.
- **Design owns the project layer** — a fourth lifecycle nudge beside runtime-probe/observability/lint-regime. Greenfield seeds it during lifecycle shaping; brownfield mines it from comprehension (unusual / opinionated / tribal patterns), **each rule human-confirmed**. Layout: `docs/standards/<topic>.md` — one concept per file, lead with the rule, code example — plus an `index.md` of one-line descriptions the Plan agent matches against. **Shrink-over-time doctrine:** when the codebase demonstrably exemplifies a rule, retire it; surrounding code teaches better than prose.
- **Validate consumes the craft layer on a second axis.** Conformance judges a **spec axis** (contract/acceptance fidelity) and a **standards axis** (baseline catalog + exactly the standards the task injected), never merged. The validator stays flags-not-fixes: standards findings (file:line, named smell/standard, direction) batch into a **[[remediation brief]]**; the orchestrator appends **one refactor task** to the feature's plan artifact (footprint = flagged files; acceptance = findings addressed or individually rebutted with evidence) executed by a fresh build agent on the feature branch, then re-validates. **Hard bound: one remediation round per feature.** Findings that survive round two are recorded advisory or park per the deviation-severity axis — never a second refactor pass. The bound is also the anti-gaming posture: a builder never iterates repeatedly against the validator's signal.
- **Scheduling: a walking-skeleton feature (`craft-baseline`), hand-built before self-hosting** — so every feature the loop builds for itself is built under the craft regime. The Design-nudge and validator-axis integrations land as notes on the `design` and `validate` nodes and are built with those features.

**Why.** The two-filesystem split (ADR-0002) already dictates the layering: shipped opinion is engine-side and swappable (hence a port, per the grilling-port precedent of binding user-level packs); project knowledge is a durable target-repo artifact. Plan-time selection reuses machinery the loop already has — the task contract is exactly the field-improvised `orchestration.yml`, but persisted, checked, and calibratable. The bounded send-back sits deliberately between the two failure modes the survey documents: advisory-only craft never gets fixed, while unbounded review loops manufacture over-engineering ("tests for cases that can't happen") and escalation fatigue.

**Considered and rejected.** Fixed (non-port) plugin content (hard-codes craft opinions into the non-swappable core, which is reserved for engine + gates); per-topic ports (three contracts for one coherent opinion set; packs don't decompose that way); a single mega-skill (couples three consumers to one artifact; build context pays for review's catalog); preload-everything (the measured bloat tax, every task); builder self-selection (invisible, unauditable, undertriggers); configure-step-owned standards (durable project knowledge belongs in the artifact spine, and Design — not Configure — shapes project judgment); emergent-only standards (no cold start; promotion-from-escalations can layer on later); advisory-only craft findings (never fixed); validator-fixes-inline (grades its own authoring — breaks ADR-0013 independence); unbounded remediation rounds (cost, over-engineering spiral, gaming pressure); deferring the bundle to self-hosting (the first self-built wave — including the bundle itself — would be built craft-blind, exactly when the bundle's value peaks).
