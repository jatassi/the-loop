# define — Define skill (interview → brief)

**Status:** designed (functional since v1, hand-built; never loop-validated).

## What it is

The first human-gated phase: get the messy idea out with the lowest capture tax,
then pressure-test it adversarially before a token is spent building. The skill
(skills/define/SKILL.md) drives the bound interview skill one question at a time,
recommended-answer style, and writes `docs/briefs/brief.md` — Decided / Deferred /
Done-looks-like sections sharp enough to design against.

## Acceptance

- A brain-dump is whittled to a structured, actionable brief.

## Notes

- Define's output never changes orientation mode — a brief only moves onboarding's
  resume point past Define (unconfigured still routes to Design next).
